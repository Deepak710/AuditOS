# AuditOS Development Workflow

Every GitHub issue follows exactly this workflow.

## Step 1

Read the GitHub Issue completely.

---

## Step 2

Read every relevant document before writing code.

This includes architecture, UX, implementation guides, routing, components and workspace documentation.

Never assume.

---

## Step 3

Determine whether existing components or layouts can be reused.

Reuse first.

Create new components only when necessary.

---

## Step 4

Implement only the requested GitHub Issue.

Do not expand scope.

Do not "improve" unrelated areas.

Do not perform refactoring unless explicitly requested.

---

## Step 5

Review the implementation.

Check

- consistency
- responsiveness
- accessibility
- design language
- naming
- folder placement
- coding standards

---

## Step 6

Update documentation.

If implementation changed

- routing
- architecture
- components
- UX
- implementation guide

update those documents.

Do not leave documentation outdated.

---

## Step 7

Prepare a completion summary.

The summary should contain

### Completed

...

### Files Created

...

### Files Modified

...

### Documentation Updated

...

### Anything requiring review

...

---

## Step 8

Stop.

Never continue into another GitHub Issue.

Never start another task automatically.

Wait for human review.
