# AuditOS Project Constitution

## Mission

AuditOS is an AI-native operating system for assurance engagements.

The current objective is NOT to build AI.

The objective is to build a world-class static prototype that validates the product architecture.

---

## Current Phase

Release 1

Static Prototype

---

## Technology Constraints

- Static HTML
- CSS
- JavaScript
- Bootstrap (local only)
- No React
- No Vue
- No Angular
- No Node.js
- No npm
- No backend
- No database
- No build tools
- Must run by double-clicking index.html on any Windows machine.

---

## Design Philosophy

Every screen must feel like part of one application.

Never mix different design systems.

Use modern enterprise SaaS UX inspired by products like Linear, Notion, Vercel, Stripe and GitHub.

21st.dev, Flowbite, Tailwind UI and shadcn are inspiration only.

Create and reuse one cohesive AuditOS design language.

---

## Development Rules

Always reuse existing layouts.

Always reuse existing components.

Never duplicate UI.

Never invent architecture that contradicts repository documentation.

If documentation conflicts,
stop and explain.

---

## Documentation Rules

Whenever implementation changes architecture,
navigation,
components,
routing,
or UX,

update the corresponding documentation before considering the issue complete.

---

## GitHub Workflow

One GitHub Issue

↓

One implementation

↓

Self-review

↓

Documentation update

↓

Human review

↓

Issue closed

Never begin another issue automatically.

---

## AI

AI is out of scope.

Any AI functionality must be mocked.

Do not implement Copilot Studio,
Power Automate,
LLMs,
memory,
agents,
or backend services.

Only create realistic placeholders.
